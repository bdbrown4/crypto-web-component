Quick Start

No build tools required — works with a plain script tag.

Option A — CDN (no install)

html
<!-- 1. Load the script (once, anywhere in your HTML) -->
<script src="https://unpkg.com/@bdbrown4/crypto-web-component/dist/crypto-web-component.js"></script>

<!-- 2. Use the tag -->
<crypto-web-component></crypto-web-component>

Option B — npm + bundler

sh
npm install @bdbrown4/crypto-web-component
js / ts — entry file
// Import once — registers the element globally for the entire app
import '@bdbrown4/crypto-web-component';
⚡ The import side-effect calls customElements.define() once and the element is available everywhere in your app from that point on — no props, no wiring, no context needed.

Framework Integration

Web Components are a browser standard — they work natively in every major framework.

html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My App</title>
</head>
<body>

  <script src="https://unpkg.com/@bdbrown4/crypto-web-component/dist/crypto-web-component.js"></script>

</html>
ts — src/main.tsx
// Register once in your entry file — before React renders
import '@bdbrown4/crypto-web-component';
import React from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';

createRoot(document.getElementById('root')!).render(<App />);
tsx — src/App.tsx
export default function App() {
  return (
    <main>
      {/* JSX accepts any valid HTML element name */}
      <crypto-web-component></crypto-web-component>
    </main>
  );
}
ℹ️ If TypeScript raises "Property does not exist on type JSX.IntrinsicElements", add a declaration to a *.d.ts file in your project:

declare namespace JSX {
  interface IntrinsicElements {
    'crypto-web-component': React.HTMLAttributes<HTMLElement>;
  }
}
ts — src/main.ts
import { createApp } from 'vue';
import '@bdbrown4/crypto-web-component';
import App from './App.vue';

const app = createApp(App);
// Tell Vue's compiler to treat hyphenated tags as custom elements
app.config.compilerOptions.isCustomElement = (tag) => tag.includes('-');
app.mount('#app');

Theming

Two complementary APIs let you fully control the component's appearance from outside its shadow DOM.

CSS Custom Properties — colors & backgrounds

CSS custom properties pierce the shadow DOM. Use these tokens to set background and text colors consistently — the floating input label will automatically match its parent panel.

PropertyDefaultControls
--cwc-container-bgantiquewhiteOuter card background
--cwc-form-bg#fffInner panel + input + floating label background
--cwc-input-color#333Input text and floating label text color
--cwc-border-color#cccInput border color (unfocused)

CSS Parts — structure & layout

Use ::part() for border-radius, padding, box-shadow, and other structural properties.

Part nameElementWhat to target
container<div>Outer card — border, border-radius, box-shadow
form<div>Inner panel — border-radius, padding
input<input>Input field — width, border-radius
button<button>"Fetch USD Value" button — background, border-radius, font-weight
result<p>Displayed price or error message — color, font-size

Dark / Purple theme — example

css
/* CSS custom properties handle background + text colors */
crypto-web-component#dark {
  --cwc-container-bg: #1a1d2e;
  --cwc-form-bg:      #12141f;
  --cwc-input-color:  #e6edf3;
  --cwc-border-color: #2e3250;
}

/* CSS Parts handle structural + accent overrides */
crypto-web-component#dark::part(container) {
  border: 1px solid #2e3250;
}
crypto-web-component#dark::part(button) {
  background: #6c63ff;
}
crypto-web-component#dark::part(result) {
  color: #6c63ff;
}

Development

The entire component lives in a single TypeScript file with no runtime dependencies.

sh
# Clone and install dev tools
git clone https://github.com/bdbrown4/crypto-web-component.git
cd crypto-web-component
npm install

# Start dev server with HMR at http://localhost:8080
npm start

# Run tests (jest + happy-dom)
npm test

# Production build → dist/crypto-web-component.js
npm run build
